## 关键实现细节

### notifications 表结构

```sql
CREATE TABLE notifications (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES profiles(id),
  firm_id UUID NOT NULL REFERENCES firms(id),
  type TEXT NOT NULL, -- 'upload', 'reminder', 'overdue', 'dormant', 'sync_failure', 'system'
  title TEXT NOT NULL,
  message TEXT NOT NULL,
  metadata JSONB, -- 存储额外数据如 client_id, upload_id 等
  is_read BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_notifications_user_unread ON notifications(user_id, is_read) WHERE is_read = FALSE;
```

### activity_logs 表结构

```sql
CREATE TABLE activity_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  firm_id UUID NOT NULL REFERENCES firms(id),
  client_id UUID REFERENCES clients(id),
  actor UUID REFERENCES profiles(id),
  type TEXT NOT NULL, -- 'upload', 'email', 'reminder', 'client_added', 'status_change'
  description TEXT NOT NULL,
  metadata JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_activity_logs_firm ON activity_logs(firm_id, created_at DESC);
```

### Resend 邮件发送激活

在 `app/api/cron/send-reminders/route.ts` 中：

```typescript
// 原代码（第 50-53 行）:
console.log(`[Send Reminder] Would send ${reminder.template} to ${client?.portal_email}`)

// 修改为:
await sendReminderEmail({
  to: client.portal_email,
  clientName: client.name,
  firmName: client.firms?.name || 'Your Accountant',
  deadlineType: reminder.template,
  deadlineDate: formatDeadline(reminder.request_id),
  uploadLink: `${APP_URL}/portal/${client.portal_token}`,
})

// 同时插入通知
await supabase.from('notifications').insert({
  user_id: user.id,
  firm_id: client.firm_id,
  type: 'reminder',
  title: 'Reminder Sent',
  message: `${client.name}: ${reminder.template} reminder sent`,
  metadata: { client_id: client.id },
})
```

### 通知中心实时获取

在 `components/dashboard/header.tsx` 中，将硬编码的 `3` 替换为：

```typescript
const [unreadCount, setUnreadCount] = useState(0)

useEffect(() => {
  fetch('/api/notifications/count')
    .then(res => res.json())
    .then(data => setUnreadCount(data.count || 0))
    .catch(() => {})
}, [])
```

## 风险与缓解

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| Resend API Key 未配置 | 高 | 启动时检查，缺少时降级为 console.log |
| 通知表迁移影响现有数据 | 中 | 先做备份，migration 使用 ADD COLUMN IF NOT EXISTS |
| Stripe webhook 重复事件 | 中 | 使用事件 ID 做幂等处理 |
| 大量通知影响性能 | 低 | 添加索引，分页获取 |
